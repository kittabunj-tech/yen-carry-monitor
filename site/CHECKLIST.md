## ✅ ROW 4 — ALERT LOG + METHODOLOGY

### Alert Log
- ตอนนี้ `alerts: []` → ต้องเห็นกล่องเส้นประ
  "ยังไม่มี alert — ระดับ CUSI ยังไม่เปลี่ยน (ระบบส่ง alert จะเริ่มทำงานใน Phase 5)"
- เมื่อ Phase 5 เติม alerts เข้ามา: การ์ดจะมีแถบสีซ้ายตามระดับ + timestamp + เหตุผล

### Methodology (พับได้)
- คลิกหัวข้อแล้วกางออก, ลูกศรหมุน 180°
- สูตร `CUSI = 50 + (weighted_z / 3) × 50`
- **ตารางน้ำหนักครบ 9 ตัว** รวม MOVE 5% ที่กำกับว่า "(ไม่มีข้อมูลวันนี้)"
- ตารางระดับ 4 สี พร้อม swatch สีและช่วงตัวเลข `0–40` `40–60` `60–80` `80–100`
- หัวข้อ "ข้อควรรู้" 4 ข้อ (lookahead, re-normalization, JGB, real-time vs computed)

> น้ำหนักและระดับทั้งหมด**อ่านจากไฟล์ JSON** ไม่ได้ hardcode ในหน้าเว็บ —
> แก้ `config.yaml` แล้วรัน pipeline ใหม่ หน้าเว็บจะเปลี่ยนตามเอง

## ⚠️ ข้อควรรู้ก่อน deploy (Phase 5)

หน้าเว็บลองหา `data/` ตามลำดับ `../data/` → `data/` → `./data/` → `/data/`
จึงใช้ได้ทั้งสองแบบ:

* **เสิร์ฟจาก repo root** (วิธีทดสอบ local) → เจอที่ `../data/`
* **GitHub Pages ที่ตั้ง root = `/site`** → ต้อง**คัดลอก `data/` เข้าไปใน `site/`**
  ตอน deploy ไม่งั้นจะไม่เจอ (spec §5 Phase 5 ข้อ 5 เตือนเรื่องนี้ไว้)

ทางที่ปลอดภัยที่สุดคือ deploy จาก **root ของ repo** แล้วตั้ง Pages ให้ใช้ `/`
โดยมี `index.html` ที่ root redirect ไป `site/` — ค่อยตัดสินใจตอน Phase 5
